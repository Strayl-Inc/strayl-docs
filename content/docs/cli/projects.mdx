## st init

Create a new Strayl project from the current directory.

```bash
st init
```

This will:
1. Ask for a project name
2. Create the project in your Strayl account
3. Upload the current directory to the sandbox
4. Create a `.strayl` config file linking this directory to the project

### Options

| Flag | Description |
|------|-------------|
| `--name <name>` | Project name (skips interactive prompt) |
| `--framework <name>` | Framework hint (`next`, `vite`, `remix`, etc.) |
| `--private` | Create as private project (default: private) |

### Example

```bash
cd my-app
st init --name "My App" --framework next
```

## st link

Link an existing local directory to an existing Strayl project.

```bash
st link
```

This creates a `.strayl` file in the current directory. If you already have a `.strayl` file (e.g., committed by a teammate), running `st link` will validate and refresh the link.

### Options

| Flag | Description |
|------|-------------|
| `--project <id>` | Project ID to link to (skips interactive selection) |

### Example

```bash
cd my-app
st link --project proj_abc123
```

## st clone

Clone a Strayl project to a local directory.

```bash
st clone <project-name-or-id>
```

This creates a new directory with the project name, downloads the current sandbox state, and creates a `.strayl` link file.

### Example

```bash
st clone my-app
cd my-app
npm install
npm run dev
```

### Options

| Flag | Description |
|------|-------------|
| `--dir <path>` | Target directory (default: project name) |
| `--branch <branch>` | Clone a specific git branch |

## st status

Show the sync status between your local directory and the Strayl sandbox.

```bash
st status
```

**Output:**

```
Project: my-app (proj_abc123)
Sandbox: synced
Local changes: 3 files modified
Last push: 2 minutes ago
Active deployment: https://my-app.strayl.app (deployed 1 hour ago)
```

### What "synced" means

- `synced` — local and sandbox are identical
- `ahead` — local has changes not in the sandbox (run `st push`)
- `behind` — sandbox has changes not locally (run `st pull`)
- `diverged` — both have changes; you'll need to merge
